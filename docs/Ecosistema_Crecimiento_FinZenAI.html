Punto de partida

Qué ya existe · qué falta construir

La buena noticia: gran parte de la plomería ya está en el FinZen real. El agente no parte de cero.

✓ Ya existe

  • ✓KPIs y DashboardPulso, Detalles, Adquisición
  • ✓Mensajería + medición de activaciónBroadcast con lift vs. holdout (efecto causal)
  • ✓Atribución externaUTM, CAC, costos e inversión por campaña
  • ✓Precedente conversacional: ZenioIA + function calling sobre la BD
  • ✓Reportes que el agente leeHoy ya se hace manual: PDF → hilos con IA

△ Falta construir

  • △API de KPIs estructuradaPara que el agente lea datos limpios, no PDFs
  • △Capa semántica de segmentosSegmentos seguros y curados, en vez de SQL libre
  • △Lectura/escritura con MetaHoy solo se envían conversiones; falta crear y leer campañas
  • △Orquestación del agente + skillEl loop, la automedición y la aprobación humana
El plan

Tres fases, de menor a mayor riesgo

Empezar por lo interno, donde ya está casi todo y no se gasta dinero externo. Conectar Meta solo cuando el bucle interno funcione.

FASE 1

Agente interno

Riesgo bajo · empezar aquí

Lee KPIs por API, conversa con los socios, encuentra segmentos y propone campañas de mensajería. Humano aprueba → se envía con el broadcast que ya existe → mide activación. Bucle cerrado, sin gasto externo.

FASE 2

Agente externo (Meta)

Riesgo medio · dinero real

Propone plan de marketing y contenido → humano aprueba → ejecuta vía la Meta Marketing API → mide CAC/ROI con la atribución que ya existe. Requiere construir la integración de campañas con Meta.

FASE 3

Plataforma multi-tenant

Visión · escalar

Empaquetar el motor como plugin para que otras empresas usen su propio agente de crecimiento. Es la evolución de negocio del doc de viabilidad ya existente.

Cómo se siente en la práctica

El agente conversacional con los socios

Un socio pide un patrón en lenguaje natural; el agente lo busca de forma segura y propone campaña interna + contenido externo sobre el mismo tema.

Socio
Búscame la gente que tiene su presupuesto pasado.
Agente
Encontré 1,240 usuarios con al menos un presupuesto excedido este mes (sin exponer datos personales). Te propongo dos acciones sobre ese segmento:
🟢 Campaña interna

Broadcast al segmento: “Tu presupuesto de Comida se pasó. Zenio te ayuda a reajustarlo en 10 segundos.” — mide el lift de activación.

🔵 Contenido externo

3 conceptos de carrusel para Instagram / TikTok: “Qué significa tener tu presupuesto en control” — para nutrir adquisición.

→ El socio revisa y aprueba antes de que se envíe o publique nada.
Lo que no se puede ignorar

Riesgos y guardarraíles

Un agente que actúa sobre dinero real y sobre toda la base de usuarios necesita límites firmes.

01

Autonomía con dinero real (Meta)

Un agente publicando ads sin control puede quemar presupuesto. Guardarraíl: aprobación humana obligatoria, topes de presupuesto y kill switch.

02

Envíos masivos autónomos

El broadcast manda push real a toda la base. Guardarraíl: aprobación + límites de frecuencia por usuario.

03

Búsqueda en la BD operacional

Darle SQL libre sobre producción arriesga performance, PII y segmentos equivocados. Guardarraíl: capa semántica de segmentos curados, solo lectura, sobre una réplica.

04

Calidad de la medición

El agente se automide contra los KPIs; si están sucios, optimiza hacia el lugar equivocado. Guardarraíl: datos limpios y estructurados por API, no PDFs ni métricas ambiguas.

Cómo se construye · handoff al pasante

El agente como proyecto independiente

Un pasante puede construirlo entero sin tocar el código de FinZen ni la base de datos. Todo pasa por una Agent API con API Key. El agente vive en su propio repo y deploy, y se conecta a cinco mundos, cada uno por su vía.

El contrato entre el pasante y FinZen
Una API Key + 3 endpoints. Todo lo demás es su mundo.

El pasante avanza solo

  • El cerebro del agente (Claude + tool-use + el loop del goal)
  • La conexión con Drive: leer el Cerebro y guardar contenido
  • La interfaz conversacional con los socios
  • La integración con Meta (ads y contenido)
  • Toda la lógica de análisis y diseño de campañas

FinZen construye una vez

  • GET /api/agent/kpis — datos estructurados
  • GET /api/agent/segments — capa semántica curada
  • POST /api/agent/campaigns — borradores
  • Cura y comparte la carpeta Cerebro con la cuenta de servicio
  • Emite la API Key + su documentación
🔑

La API Key del pasante va con permisos mínimos: solo lee KPIs/segmentos y crea borradores. No puede enviar campañas, no ve datos personales, no toca finanzas ni la base directamente. Es rate-limited y revocable — si algo sale mal, se apaga sin afectar a FinZen.
